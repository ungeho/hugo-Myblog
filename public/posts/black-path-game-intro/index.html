AI補助で開発したブラウザゲーム「Black Path Game」

ゲーム画面#

現在の画面は、中央に盤面、上に HUD、右に設定パネル という構成です。
ルールの追加で情報量は増えていますが、できるだけ「いま何を選べるか」が崩れないようにまとめています。

どんなゲーム？#

Black Path Game は、AI補助で開発した、ブラウザで遊べる 2 人対戦のパス構築ゲームです。
大元の発想は Black 系のルールに近いのですが、そのまま再現するのではなく、欠損マス・罠・制限時間 を加えることで、読み合いの性質がかなり変わるようにしています。

自分の中では、単なる再現よりも

  • ルールが分かりやすいこと
  • 対戦ゲームとして読み合いが生まれること
  • AI 相手でも人相手でも成立すること

を大事にして作っています。

Black Path Game とは？#

元になっている Black Path Game は、1960 年に Larry Black によって考案されたとされる 2 人用の盤上ゲームです。
道を伸ばしていき、自分の手で経路を盤外へ出してしまった側が負けるという考え方が核になっています。

今回の実装は、その発想を借りつつも、

  • 右下固定 + ランダム追加の欠損マス
  • 隠し情報としての罠
  • 短い持ち時間

を加えているので、元の Black をそのまま移植したものというより、Black 系のアイデアを土台にした対戦寄りのバリアント と考えるのが近いです。

何をするゲームなのか#

盤面上では、道の先端に続くマスへタイルを置いて、パスを伸ばしていきます。
左上のスタート地点には最初から十字タイルが置かれていて、そこから経路が進んでいきます。

基本の感覚としては、

  • 道の先端にタイルを置く
  • 既にあるタイルには自動追従する
  • 自分の手で道を危険な場所へ押し出してしまうと負ける

という流れです。

ただし、このゲームでは 右下の固定欠損マス が最初からあり、さらに設定によってランダムな欠損マスを追加できます。
そのため、盤外だけでなく「どこが穴になっているか」もかなり重要になります。

このバリアントで大きい 3 つの要素#

欠損マス#

このゲームでは、右下のマスが常に欠損しています。
さらに設定で、指定した数だけランダムな欠損マス を追加できます。

つまり、毎回同じ形の盤面で遊ぶだけでなく、

  • 穴の数を増やして盤面を荒らす
  • 毎回違う盤面の危険地帯を読む
  • 終盤の逃げ道を減らす

といった遊び方ができます。

単に難しくするというより、安全そうに見える経路が途中で急に危険になる感覚 を作れるのが面白いところです。

罠#

このバリアントでいちばん特徴的なのは、やはり 罠 です。

各プレイヤーは、設定した個数ぶんだけ任意のマスに罠を置けます。
罠の効果は、「そのマスで選べる 3 種類のタイルのうち 1 つを潰す」ことです。

つまり、そのマスに到達したときの選択肢が減ります。

同じマスに罠が重なったとき
  • 両者が同じマスで同じタイルを潰した
  • 両者が同じマスで別々のタイルを潰した
その結果どうなるか
  • 選択肢は 2 つのまま
  • 選択肢は 1 つまで減る

この仕様があるので、罠は「ただの禁止マス」ではなく、将来そのマスに来たときの分岐をどこまで細くするか という形で効いてきます。

しかも、自分が置いた罠は見えますが、相手がどこに置いたかは見えません。
そのため、盤面上の経路だけでなく、見えていない情報も含めて読む必要があります。

制限時間#

さらに、1 手あたりの持ち時間を設定できます。
現行版では なし / 3s / 5s / 7s / 9s から選べます。

時間制限があると、

  • 罠の位置を読んで悩みすぎると負ける
  • 終盤の強制手で判断が雑になる
  • 対人戦でも AI 戦でもテンポが変わる

ので、盤面パズルというより 対戦ゲームとしての緊張感 がかなり強くなります。

画面と操作#

現行版の画面は、大きく分けると

  • 盤面
  • 上部の HUD
  • 右側の操作 / ゲーム設定パネル

で構成しています。

右側では、

  • 対人戦 / vs AI
  • Easy / Medium / Hard
  • 先手 / 後手
  • 盤面サイズ
  • 欠損マス
  • 罠
  • 制限時間

を切り替えられるので、1 本のアプリの中で遊び味をかなり変えられます。

盤面側では、

  • 合法手のハイライト
  • 経路の可視化
  • 直前の流れの追跡
  • 罠配置フェーズでの自分の罠確認

ができるようにしていて、ルールが増えても盤面の読みやすさが落ちすぎないようにしています。

vs AI でも遊べる#

このゲームは 対人戦 だけでなく vs AI にも対応しています。
難易度は Easy / Medium / Hard の 3 段階です。

AI 戦があることで、

  • ルール確認だけしたいとき
  • 新しい設定を試したいとき
  • 罠あり盤面の感覚を練習したいとき

にも 1 人で触りやすくしています。

どういう読み合いが生まれるか#

このゲームが面白いのは、単に「今どこへ伸びるか」だけではなく、

  • 欠損マスがどこにあるか
  • その先で罠に選択肢を削られるか
  • 時間内にどこまで読めるか

を一緒に考える必要があるところです。

  1. まず見えている経路の危険度を読む
  2. 次に、欠損マスで詰まる形を警戒する
  3. さらに、見えていない相手の罠を想定する
  4. 制限時間があるなら、その読みを短時間で決める

この多層構造があるので、見た目はシンプルでも、実際の対戦ではかなりいやらしいゲームになります。

技術的な構成#

技術的には React + TypeScript + Vite ベースのブラウザゲームです。

UI 側
  • React
  • TypeScript
  • Vite
  • CSS Modules
ゲームロジック側
  • 盤面状態
  • 合法手生成
  • ゲーム進行
  • 罠配置
  • AI 思考

実装としては、UI とルール処理を分けていて、

  • engine.ts
  • moveGenerator.ts
  • rules.ts
  • ai.ts

あたりに責務を分離しています。
README が古くなっていても、コード側は「今のルールがどこにあるか」を追いやすい構成にしてあります。

どういう人に向いているか#

  1. シンプルな見た目で読み合いが深いゲームが好き
  2. 隠し情報や罠で分岐が絞られる感覚が好き
  3. AI 戦と対人戦の両方を 1 本で遊びたい
  4. オリジナル寄りの Black バリアントを触ってみたい

逆に、定番ルールをできるだけ忠実に再現したものを探している場合とは少し方向性が違います。
このゲームは、Black 系の発想を土台にしつつ、対戦用にかなり味付けした版 と捉えるのが近いです。

FAQ#

元の Black と同じゲーム？
完全に同じではありません。
大元の発想は近いですが、欠損マス、罠、制限時間などを入れていて、かなり別の読み合いになっています。
罠はどう見える？
自分が置いた罠の場所と内容は確認できますが、相手が置いた罠は見えません。
そのため、対戦では見えていない情報を含めた読みが必要になります。
同じマスに両者が罠を置いたら？
同じタイルを潰していた場合は選択肢は 2 つのまま、別々のタイルを潰していた場合は選択肢が 1 つまで減ります。

まとめ#

このアプリでやりたかったこと
  • Black 系の発想をベースにしつつ、別の読み合いを持つゲームにすること
  • 欠損マス、罠、制限時間で盤面の緊張感を増やすこと
  • 対人戦でも AI 戦でも遊べる形にすること
  • ルール追加後も盤面が分かりやすい UI にすること

Black Path Game は、もとの発想を借りつつ、自分なりに対戦ゲームとして面白くなる方向へ崩したバリアント です。
欠損マス、隠し情報としての罠、短い持ち時間が重なることで、見た目以上に緊張感のあるゲームになっていると思います。
