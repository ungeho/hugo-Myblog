どんなアプリ？#

Budget Book は、できるだけ素直に使えることを意識して作った家計簿 Web アプリです。
日々の収支を記録するだけでなく、あとから見返したときに「何に使っているのか」「予算をどれくらい使っているのか」「固定費がどれくらいあるのか」が分かるところまでまとめています。

作るときに意識したのは、次の 3 つです。

  • 入力が面倒すぎないこと
  • 集計が見て分かりやすいこと
  • スマホでも PC でも扱いやすいこと

何ができるのか#

収支記録と一覧管理

基本の機能は、家計簿として必要なところを一通り揃えています。

  • 日付、収入 / 支出、カテゴリ、金額、メモを入力して記録
  • 一覧表示、編集、削除
  • 月別フィルタで過去の記録を確認
  • 収入 / 支出の両方に対応

単に「入力して終わり」ではなく、あとから見返す前提で、一覧の編集や月ごとの切り替えも自然に使えるようにしています。

集計と可視化

家計簿アプリを作るときに、個人的にかなり大事だと思っていたのがこの部分です。

  • 月次サマリー
  • カテゴリ別内訳
  • ドーナツチャート
  • 直近 6 ヶ月の月別棒グラフ
  • 支出増加のアラート

金額を並べるだけだと見返しづらいので、「今月どうだったか」と「何に偏っているか」が一目で分かることを意識しています。

月次サマリー#

月次サマリーでは、

  • 収入合計
  • 支出合計
  • 収支
  • サブスク月額

をまとめて確認できます。
最初にここを見るだけで、その月の状態をかなり把握しやすくしています。

カテゴリ別の見え方#

カテゴリ別内訳とドーナツチャートは、「何に使っているか」を見やすくするために入れました。
支出が多いカテゴリが埋もれないようにしているので、見返したときに無駄遣いの傾向も拾いやすいです。

月別の推移#

単月の数字だけだと変化が見えづらいので、直近 6 ヶ月の収入 / 支出推移も棒グラフで確認できるようにしています。
このあたりは「今月だけを見るアプリ」ではなく、「少しずつ見返して使うアプリ」にしたかった部分です。

予算管理とサブスク管理も入れた理由#

月間予算#

カテゴリごとに月間予算を設定できるようにしています。

  • 予算額の設定
  • 消化率のプログレス表示
  • 80% 以上で警戒
  • 超過時は赤系で表示

家計簿は記録だけだと、どうしても「見たけど何もしない」になりやすいので、予算に対して今どれくらい使っているかが自然に見えるようにしました。

サブスクリプション管理#

固定費の把握も大事なので、サブスク管理も別枠で入れています。

  • 月額 / 年額の登録
  • 月額換算コストの表示
  • 定期支出の一覧管理

年額サービスも月額換算で見えるようにしているので、「気づいたら固定費が増えていた」を見つけやすくしています。

無駄遣いアラート#

前月比で 30% 以上支出が増えたカテゴリを検出する仕組みも入れています。

これは厳密な家計分析というより、「先月より急に増えてない？」を早く見つけるための軽いアラートです。
細かい分析を毎回しなくても、変化が大きいところを先に見つけられるようにしたかったので入れました。

画面構成#

PC では 2 カラム#

PC では、

  • 左に入力フォーム、サブスク、CSV 入出力
  • 右に集計、グラフ、一覧

という 2 カラム寄りの構成にしています。

入力しながら右側の集計も見えるので、画面を行き来しすぎずに使えるようにしています。

モバイルではタブ切り替え#

スマホではそのまま 2 カラムにすると厳しいので、

  • 入力
  • 集計
  • 明細

の 3 タブに切り替わるようにしています。
レスポンシブ対応というより、モバイルでは使い方自体を少し変える方向で作っています。

技術的な構成#

README にもある通り、構成はかなり素直です。

フロントエンド
  • React 19
  • TypeScript (strict)
  • Vite 8
  • SVG ベースのチャート
バックエンド
  • Express 5
  • TypeScript
  • SQLite
  • better-sqlite3

個人的には、家計簿のようにローカルで持ちたい情報を扱うアプリは、 軽くて扱いやすい構成 が相性いいと思っています。
その意味でも、SQLite を使ったシンプルな構成はかなりしっくりきました。

セットアップ#

起動方法もそこまで複雑ではありません。

Bash
npm install
npm run dev

これで、

  • サーバー: http://localhost:3001
  • Vite dev server: http://localhost:5173

が立ち上がります。
本番ビルド後は npm start で 3001 を開く形です。

同じネットワーク内の他端末からもアクセスできるので、自分の PC だけでなく家の中のスマホから使うことも想定できます。

どういう人に向いているか#

  1. 細かすぎない家計簿を Web で持ちたい
  2. 収支記録だけでなく、カテゴリ集計や予算も見たい
  3. サブスクの固定費も一緒に把握したい
  4. PC とスマホの両方で使いたい

逆に、銀行連携や自動仕訳、資産管理のような大きな機能が主目的だと、今の Budget Book は少し方向性が違うと思います。

FAQ#

CSV の入出力はできる？
できます。
CSV エクスポートと CSV インポートの両方に対応していて、README では Excel で扱いやすい BOM 付き UTF-8 のエクスポートにも触れています。
カテゴリは増やせる？
はい。デフォルトカテゴリに加えて、カスタムカテゴリを追加できます。
データはどこに保存される？
SQLite を使っていて、data/budget-book.db に保存されます。
初回起動時にテーブルとデフォルトカテゴリを自動作成する構成です。

まとめ#

このアプリでやりたかったこと
  • 毎日の収支を素直に記録できること
  • 集計やグラフで見返しやすいこと
  • 予算やサブスクの固定費まで一緒に把握できること
  • PC とスマホの両方で使いやすいこと

Budget Book は、家計簿として必要なことを一通り押さえつつ、 「あとから見返したときに分かりやすいこと」 を大事にして作ったアプリです。
複雑すぎる家計管理ツールではなく、日常的に続けやすい Web 家計簿として育てていきたいと思っています。
