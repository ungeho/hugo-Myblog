## どんなゲーム？

`Dodgem Browser` は、**AI補助で開発した、ブラウザ上で遊べるドッジェムゲーム**です。  
昔ながらのルールをベースにしつつ、そのまま再現するだけでなく、**遊びやすさと見返しやすさ** をかなり大事にして作っています。

作るときに意識した軸は、次の 3 つです。

- まず普通に遊びやすいこと
- CPU やオンライン対戦まで含めて 1 本で完結すること
- ルールロジックを後から触りやすいこと

## 何ができるのか

### 3 つの対戦モード

このゲームは、単純に盤面だけ置いた実装ではなく、遊び方を複数用意しています。

- 人対人
- CPU対戦
- オンライン対戦

CPU 対戦だけでも遊べますし、ローカルタブ対戦や WebSocket 経由のリモート対戦にも対応しているので、**1 人で試すところから人と遊ぶところまで** ちゃんとつながる形にしています。

### 対局を追いやすくする機能

対戦まわりでは、見た目とログの両方で状況を追いやすくしています。

- 合法手のハイライト
- 選択中の駒の表示
- 直前の手の表示
- 移動経路の可視化
- Undo
- 対局ログ
- リプレイ

ただ盤面を出すだけでなく、**あとから局面を見返したり、何が起きたか追えること** をかなり重視しています。

### ルールバリエーション

標準ルールだけでなく、バリエーションも入れています。

- クラシック 4x4
- ワイド 5x5
- スプリント 5x5

これによって、同じゲームでも盤面の広さやテンポ感の違いを試しやすくしています。

## ルール実装で意識したこと

README にも書いてある通り、この実装では **曖昧になりやすい箇所に対してシンプルで一貫したローカルルール** を採用しています。

{{< compare left="ルール実装で大事にしたこと" right="そのためにやったこと" >}}
- あいまいな裁定を減らしたい
- 盤面上で迷わず遊べるようにしたい
- ルール処理をコードとして読みやすくしたい
<!--split-->
- `rules.ts` にロジックを集約
- 合法手と勝敗判定を明確化
- バリエーション差分を別定義に分離
{{< /compare >}}

特にこの手のゲームは、盤面よりも **ルールの境界条件** で挙動がぶれやすいので、そこを UI 側に散らさず、ゲームロジック側へまとめることを意識しました。

## CPU とオンライン

CPU 側は `random`、`heuristic`、`lookahead` の 3 戦略を持っていて、難易度としては

- かんたん
- ふつう
- むずかしい

の 3 段階で遊べるようにしています。

オンラインは 2 系統あります。

- 同じブラウザ・同一オリジンで使う `BroadcastChannel` ベースのローカルタブ対戦
- WebSocket サーバー経由で別端末とつなぐリモート対戦

この分け方にしているのは、**オンライン機能を試すハードルを下げたかった**からです。  
まずはローカルで試せて、必要ならリモートにも広げられる形が扱いやすいと思っています。

## 共有と見返しやすさ

このゲームでは、共有リンクで局面や設定、対局ログを受け渡せるようにしています。

- 局面共有
- 設定共有
- 対局ログ共有
- リプレイ

単に遊ぶだけでなく、「この局面どうだったか」「このバリエーションでこう進んだ」といった情報を残しやすくしたかったので、このあたりはかなり重要な機能です。

## 技術的な構成

技術的には、かなり素直な React + TypeScript + Vite 構成です。

{{< compare left="フロント側" right="ロジック / 補助層" >}}
- React 18
- TypeScript
- Vite
- CSS
<!--split-->
- ルールロジック
- CPU 思考
- セッション / Undo
- 共有リンク
- オンライン接続 helper
{{< /compare >}}

さらに、

- `rules.ts`
- `variants.ts`
- `cpu.ts`
- `session.ts`
- `share.ts`
- `online.ts`

のように責務を分けているので、**UI とゲームロジックをある程度切り離したまま育てやすい** のが自分としてはかなり気に入っているポイントです。

## どういう人に向いているか

{{< steps >}}
1. ルールゲームをブラウザで気軽に遊びたい
2. CPU 対戦やオンライン対戦まで一通り試したい
3. 対局ログやリプレイで局面を見返したい
4. UI だけでなくゲームロジックの実装にも興味がある
{{< /steps >}}

逆に、派手な演出のあるアクション寄りゲームを求めている場合とは少し方向性が違います。  
このアプリは、**盤面ゲームとしての分かりやすさと育てやすさ** を優先しています。
