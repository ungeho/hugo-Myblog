## どんなアプリ？

`LAN Library` は、**AI補助で開発した、LAN 内だけで使うことを前提にしたローカルライブラリ / 漫画ビュワー**です。  
外部サービスに預けるのではなく、自分の PC や NAS に置いたライブラリを家の中の端末から読む、という使い方に素直に寄せています。

作るときに軸にしたのはかなりシンプルです。

- 完全ローカル前提
- LAN 内の複数端末からアクセス可能
- サーバー 1 つで家中の端末から閲覧できる

クラウド同期型の読書サービスとは違って、 **「自分の家の中で完結する本棚」** を作りたい、という発想から作っています。

## 何ができるのか

{{< section_label tone="mint" >}}取り込みと管理{{< /section_label >}}

取り込みと管理まわりは、実際に自分で使いやすいことを意識してこうしています。

- フォルダ形式と ZIP 形式の両方に対応
- `library/` 配下の作品を自動検出
- ZIP ファイルのドラッグ&ドロップ追加
- `LIBRARY_DIR` で NAS や外部ストレージへ切り替え可能
- タイトル、セクション、シリーズ、巻数、カテゴリの編集
- 一括編集
- 評価機能
- 読書状態と読了率の管理

単に **「画像を読む」だけで終わらない**ようにしたかったので、本棚として持ち続ける前提で、分類やシリーズ整理までできるようにしています。

{{< section_label tone="blue" >}}閲覧体験{{< /section_label >}}

リーダー部分も、単に画像を 1 枚ずつ出すだけではありません。

- 単ページ / 見開きの切り替え
- 右開き / 左開きの切り替え
- 全画面モード
- キーボード操作
- 画面の左右クリックでページ送り
- 中央クリックで UI 表示切り替え

細かいところでは、**読書方向を `localStorage` に保存**したり、**読書位置を保存**したり、さらに**前後ページを軽く preload**したりして、日常的に使いやすい感触を意識しています。

## 実際の画面の流れ

このアプリは、画面の責務がかなり素直になるように分けています。

### 1. 本棚画面

本棚画面では、作品一覧を中心にしつつ、次のような整理ができます。

- セクション別タブ
- タイトル / セクション / カテゴリ検索
- シリーズ絞り込み
- カテゴリ絞り込み
- 読書状態での絞り込み
- タイトル順 / 追加日順 / ページ数順 / 最近読んだ順 / 評価順のソート
- 一括編集モード

シリーズ付きの作品はシリーズ単位でまとめ、巻数で並ぶようにしています。  
単なる「画像置き場」ではなく、**あとから見返すための本棚**として使える形を目指しました。

### 2. 詳細画面

詳細画面では、作品情報を確認しながら、その場で編集できます。

- カバー表示
- 読むボタン
- セクション、シリーズ、巻数、カテゴリ表示
- 星評価
- 読書状態の変更
- その場で新しいセクション / シリーズ / カテゴリを追加
- サムネイル一覧からページ確認

ここは自分でも気に入っているところで、**メタデータ編集のために別の管理画面へ飛ばなくてよい**構成にしています。  
作品を見ながら整理できるので、ライブラリを育てやすくなっています。

### 3. リーダー画面

リーダーは、かなり「読むこと」に寄った作りです。

- `ArrowLeft / ArrowRight` で移動
- `PageUp / PageDown` でも移動
- `Space` で次ページ
- `F` で見開き切り替え
- `Esc` で UI の切り替えまたは全画面解除

さらに、右開き / 左開きの違いに応じて左右のページ送りも自然に変わるので、漫画ビューアとして欲しい配慮はきちんと入れました。

## セットアップもイメージしやすい

起動手順はかなりシンプルにしています。

```bash
npm install
npm run dev
```

- フロントエンド: `http://localhost:5173`
- API サーバー: `http://localhost:3001`
- LAN 内の他端末: `http://<ホストIP>:5173`

ライブラリ自体も、`library/` の中に作品フォルダか ZIP を置く、という分かりやすい構成です。

```text
library/
├── 作品A/
├── 作品B.zip
└── 作品C/
```

この「まず置けば見つかる」感はかなり大事だと思っていました。  
ローカル向けツールは、最初の導入で面倒さが強いと続きにくいので、その点でも入りやすくしたかったです。
